# API Specification

Base URL: `http://127.0.0.1:8000`

## Health

### `GET /health`
Returns API liveness.

## Trades

### `POST /api/trades`
Create a DRAFT trade.

Request body:
```json
{
  "direction": "BUY",
  "quantity_mw": 100,
  "price_per_mwh": 45.5,
  "counterparty": "Shell",
  "delivery_start": "2026-06-18T00:00:00Z",
  "delivery_end": "2026-06-19T00:00:00Z",
  "hub": "MISO",
  "currency": "USD",
  "notes": "optional"
}
```

### `GET /api/trades`
List trades. Optional filters:
- `status`
- `counterparty`
- `hub`

### `GET /api/trades/{trade_id}`
Get a trade by integer ID.

### `PUT /api/trades/{trade_id}`
Update a trade. Allowed only when status is DRAFT.

### `DELETE /api/trades/{trade_id}`
Delete a trade.

Notes:
- `trade_id` is an auto-incrementing integer generated by the API.

## Evaluation Fault Injection

For test/evaluation scenarios, the server intentionally injects failures across all endpoints:
- intermittent transient failures with `503 Service Unavailable` and `Retry-After: 1`
- intermittent delayed responses
- intermittent simulated timeouts with `504 Gateway Timeout`

Bypass option for deterministic automation:
- send request header `x-eval-bypass-chaos: true`
